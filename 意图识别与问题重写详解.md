### 4.1 调用位置

### 4.2 工作原理

**核心思想**: 根据意图识别结果，生成多个不同角度的检索查询，提高召回率

### 4.3 重写规则

```python
def rewrite_query(marketing_text, intent_result):
    queries = []

    # 1. 原始查询（保留原文）
    queries.append(marketing_text)

    # 2. 根据主要意图生成专业术语查询
    primary_intent = intent_result["primary_intent"]

    if primary_intent == "承诺保证收益":
        queries.extend([
            "保险产品收益保证承诺 固定收益 保本保收益",  # 专业术语
            "保险产品风险提示 收益不确定性说明",          # 反面查询
        ])
    elif primary_intent == "夸大宣传":
        queries.extend([
            "保险营销绝对化用语 夸大宣传 最高最好",
            "保险营销合规用语 宣传规范",
        ])
    # ... 其他意图类似

    # 3. 高风险时添加通用合规查询
    if intent_result["risk_level"] in ["high", "medium"]:
        queries.extend([
            "保险销售合规要求 禁止性规定",
            "互联网保险业务监管 营销宣传规范",
        ])

    # 4. 去重并返回
    return list(set(queries))
```

### 4.4 实际案例演示

**输入文案**: `"本保险产品保本保收益，年化收益率保证8%，零风险，稳赚不赔！"`

**意图识别结果**:
- 主要意图: "承诺保证收益"
- 风险等级: "high"

**问题重写过程**:
```python
# 原始查询
queries = ["本保险产品保本保收益，年化收益率保证8%，零风险，稳赚不赔！"]

# 根据意图 "承诺保证收益" 添加专业查询
queries += [
    "保险产品收益保证承诺 固定收益 保本保收益",  # 正面：监管如何描述这种违规
    "保险产品风险提示 收益不确定性说明",          # 反面：合规的风险提示长什么样
]

# 风险等级为 "high"，添加通用查询
queries += [
    "保险销售合规要求 禁止性规定",
    "互联网保险业务监管 营销宣传规范",
]

# 去重后返回
return [
    "保险产品收益保证承诺 固定收益 保本保收益",
    "保险产品风险提示 收益不确定性说明",
    "本保险产品保本保收益，年化收益率保证8%，零风险，稳赚不赔！",
    "保险销售合规要求 禁止性规定",
    "互联网保险业务监管 营销宣传规范",
]
```

### 4.5 为什么要这样重写？

| 查询类型 | 作用 | 示例 |
|---------|------|------|
| 原始查询 | 保留用户输入的完整语义 | "保本保收益8%" |
| 专业术语查询 | 用监管标准术语检索 | "收益保证承诺" |
| 反面查询 | 查询合规要求，对比判断 | "风险提示说明" |
| 通用查询 | 召回基础监管条文 | "禁止性规定" |

## 六、核心优势

### 6.1 为什么需要意图识别？

| 没有意图识别 | 有意图识别 |
|-------------|-----------|
| 只能通过关键词匹配 | 理解文案的违规类型 |
| 检索查询单一 | 生成针对性查询 |
| 召回率低 | 召回率高 |

### 6.2 为什么需要问题重写？

**场景**: 用户输入 "这个产品稳赚不赔"

| 直接检索 | 问题重写后检索 |
|---------|---------------|
| "这个产品稳赚不赔" | "这个产品稳赚不赔" |
| | "保险产品收益保证承诺" |
| | "保险产品风险提示说明" |
| | "保险销售禁止性规定" |

**效果对比**:
- 直接检索: 可能只召回1-2个相关条文
- 重写后检索: 召回5-10个相关条文，覆盖更全面
