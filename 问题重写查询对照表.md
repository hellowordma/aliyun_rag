## 三种查询类型详解

### 1️⃣ 专业术语查询（正面查询）

**作用**: 用监管标准术语描述违规行为，召回相关禁令条文

| 违规类型 | 口语化表达 | 专业术语查询 |
|---------|-----------|-------------|
| 承诺保证收益 | "稳赚不赔" | "保险产品收益保证承诺 固定收益" |
| 夸大宣传 | "我们最好" | "保险营销绝对化用语 夸大宣传" |
| 误导性宣传 | "比存款好" | "保险产品性质说明 误导混淆 理财产品" |
| 无证代言 | "明星推荐" | "保险销售资质要求 代言推荐" |
| 风险提示不足 | "肯定赚钱" | "保险产品风险提示义务 信息披露" |

**示例**:
```
原文: "本产品稳赚不赔，零风险！"
      ↓
专业术语: "保险产品收益保证承诺 固定收益 保本保收益"
      ↓
召回条文: "保险机构不得承诺收益或承诺承担损失..."
```

---

### 2️⃣ 反面查询（合规要求）

**作用**: 查询"应该怎么做"，用于对比判断

| 违规类型 | 违规做法 | 反面查询（合规要求） |
|---------|---------|---------------------|
| 承诺保证收益 | 保证收益 | "保险产品风险提示 收益不确定性说明" |
| 夸大宣传 | 说"最好" | "保险营销合规用语 宣传规范" |
| 误导性宣传 | 混淆产品 | "保险销售真实性要求 信息披露" |
| 无证代言 | 无证代言 | "保险销售人员管理 执业登记" |
| 风险提示不足 | 不提示风险 | "保险条款说明 责任免除 犹豫期" |

**示例**:
```
原文: "本产品稳赚不赔！"
      ↓
反面查询: "保险产品风险提示 收益不确定性说明"
      ↓
召回条文: "应当向投保人说明分红保险的不确定性..."
      ↓
对比判断: 原文没提示风险，违规！
```

---

### 3️⃣ 通用查询（兜底查询）

**作用**: 确保覆盖核心禁令，防止漏检

```python
# 当风险等级为 high 或 medium 时添加
if risk_level in ["high", "medium"]:
    queries.extend([
        "保险销售合规要求 禁止性规定",
        "互联网保险业务监管 营销宣传规范",
    ])
```

**适用场景**: 所有高风险违规类型

## 设计思路总结

```
┌─────────────────────────────────────────────────────────────┐
│  查询类型       │  目的                  │  召回的内容          │
├─────────────────────────────────────────────────────────────┤
│  原始查询       │  保留用户输入          │  与原文语义相近的条文  │
│  专业术语查询   │  转为监管术语          │  禁止该行为的条文      │
│  反面查询       │  查询合规要求          │  应该怎么做的条文      │
│  通用查询       │  兜底核心禁令          │  基础性监管条文        │
└─────────────────────────────────────────────────────────────┘
```

**核心思想**: 从"正面（违规描述）"和"反面（合规要求）"两个角度，用"口语"和"专业术语"两种表达，全方位检索相关条文。
