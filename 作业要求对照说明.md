#### 模型路由设计

系统根据**输入类型**自动选择合适的模型：

```
输入类型判断 (multimodal_auditor.py)
        │
        ├── 纯文本 (text)
        │       │
        │       ▼
        │   auditor.audit_marketing_text()
        │       │
        │       ▼
        │   client.chat(messages, model="qwen-plus")  ← 文本LLM
        │
        ├── 图片 (image)
        │       │
        │       ▼
        │   multimodal_auditor.audit_marketing_image()
        │       │
        │       ├── 第一步: client.analyze_marketing_image() → qwen-vl-plus (提取图片内容)
        │       │
        │       ▼
        │   client.chat(messages, model="qwen-vl-plus")  ← 多模态LLM
        │
        └── 图文混合 (multimodal)
                │
                ▼
            multimodal_auditor.audit_marketing_image()
                │
                ├── 第一步: client.analyze_marketing_image(text_context) → qwen-vl-plus
                │
                ▼
                client.chat(messages, model="qwen-vl-plus")  ← 多模态LLM
```

**核心路由代码** (`multimodal_auditor.py`):
```python
def audit_marketing_multimodal(content, kb, client, content_type="text"):
    """统一的多模态审核入口 - 根据输入类型路由"""
    if content_type == "text":
        # 纯文本 → 使用 qwen-plus
        from .auditor import audit_marketing_text
        return audit_marketing_text(content, kb, client)
    elif content_type == "image":
        # 图片 → 使用 qwen-vl-plus
        return audit_marketing_image(image_bytes=content, kb=kb, client=client)
    elif content_type == "multimodal":
        # 图文混合 → 使用 qwen-vl-plus
        text_context, image_bytes = content
        return audit_marketing_image(image_bytes, text_context=text_context, ...)
```

**底层调用** (`bailian_client.py`):
```python
# 文本聊天 - qwen-plus
def chat(self, messages, model=None):
    return self.client.chat.completions.create(
        model=model or self.settings.chat_model,  # 默认 qwen-plus
        messages=messages,
    )

# 图片分析 - qwen-vl-plus
def analyze_marketing_image(self, image_bytes, ...):
    messages = [{
        "content": [
            {"type": "text", "text": prompt},
            {"type": "image_url", "image_url": {"url": base64_image}}  # 多模态格式
        ]
    }]
    return self.chat(messages, model=self.settings.vl_model)  # qwen-vl-plus
```

### ✅ 要求2：至少使用一种增强技术（RAG / Agent / Prompt Engineering / Workflow）

本项目使用了 **RAG（检索增强生成）和 Prompt Engineering** 作为核心技术增强：

#### 2.1 RAG 技术实现

| 组件 | 实现方式 |
|------|---------|
| **知识库构建** | 向量化存储监管条文（1024维embeddings） |
| **向量检索** | 基于余弦相似度的相关条文检索 |
| **混合检索** | BM25稀疏检索 + 向量稠密检索，RRF融合 |
| **增强推理** | 检索到的条文作为上下文注入LLM Prompt |

**实现文件**: `knowledge_base.py`, `knowledge_base_milvus.py`, `hybrid_retriever.py`

**关键代码**:
```python
from aliyun_rag.knowledge_base import load_knowledge_base
from aliyun_rag import audit_marketing_text

# 1. 加载RAG知识库
kb = load_knowledge_base('kb')  # 包含137个监管条文chunks

# 2. 审核时自动检索相关条文
result = audit_marketing_text(
    marketing_text="本保险保证年化收益8%",
    kb=kb,  # RAG增强
    client=client,
    top_k=6  # 检索最相关的6条条文

# 3. 违规判断基于检索到的法规条文

```

**路由机制说明**:

1. **用户输入** → 系统接收三种输入类型
2. **模型路由层** → `audit_marketing_multimodal()` 根据 `content_type` 判断
3. **模型选择层** → 根据路径自动选择合适的模型：
   - **文本路径** → `qwen-plus` (成本更低，速度更快)
   - **图片路径** → `qwen-vl-plus` (支持视觉理解)
4. **后续处理** → 所有路径汇聚到统一的 RAG 检索和 Prompt Engineering 流程

##### (2) RAG 混合检索设计

**检索公式**:
```
Final_Score = 0.3 × BM25_Score + 0.7 × Dense_Score
```

**优势**:
- BM25：精确匹配关键词
- 向量检索：语义相似度
- 融合：结合两者优势

```python
# hybrid_retriever.py
def _fuse_results(dense_results, sparse_results):
    k = 60  # RRF常数
    fused = {}

    for i, item in enumerate(dense_results):
        fused[item['id']] = 0.7 * (1 / (k + i + 1))

    for i, item in enumerate(sparse_results):
        score = 0.3 * (1 / (k + i + 1))
        fused[item['id']] = fused.get(item['id'], 0) + score

    return sorted(fused.items(), key=lambda x: x[1], reverse=True)
```
