# 保险营销内容智能审核系统 - 面试演示准备

## 一、项目概述（1-2分钟开场）

### 1.1 背景问题
```
金融行业对外营销内容数量激增，传统人工审核面临：
- 效率低：人工审核耗时耗力
- 成本高：需要专业合规人员
- 风险大：难以应对复杂多变的监管要求
```

### 1.2 解决方案
```
基于阿里云百炼大模型 + RAG技术构建的智能审核系统

核心价值：
- 自动化审核，提升效率
- 精准引用监管条文
- 支持文本、图片、PDF多模态输入
- 可解释性强（输出违规理由、条文编号、置信度）
```

### 1.3 技术栈
| 组件 | 技术选型 |
|------|---------|
| 大模型 | 阿里云百炼（通义千问 qwen-plus） |
| 视觉模型 | qwen-vl-plus（图片OCR） |
| 嵌入模型 | text-embedding-v3（1024维） |
| 向量检索 | Milvus Lite / NumPy |
| 稀疏检索 | BM25 + jieba 分词 |
| 混合检索 | RRF融合（0.3 BM25 + 0.7 向量） |
| Web框架 | Gradio 4.0+ |

### 2.2 核心模块说明

#### 知识库构建 (`knowledge_base.py`)
```python
# 核心流程
1. 文档解析：从PDF/DOCX提取文本（支持多模态OCR）
2. 条文切分：按"第X条"正则匹配切分法规
3. 向量化：调用text-embedding-v3生成1024维向量
4. 持久化：保存为chunks.jsonl + embeddings.npy
```

#### 混合检索 (`hybrid_retriever.py`)
```python
# RRF (Reciprocal Rank Fusion) 融合算法
fusion_score = 0.7 × normalized_dense_score + 0.3 × normalized_bm25_score

# 为什么用混合检索？
- 稠密检索：擅长语义理解（"保证收益" ≈ "保本保息"）
- 稀疏检索：擅长精确匹配（条文编号、专业术语）
- 融合后：兼顾语义和精度，召回率提升
```

#### 多模态审核 (`multimodal_auditor.py`)
```python
# 图片审核两步走
Step 1: 图片内容提取
  - 使用qwen-vl-plus提取图片中的文字
  - 识别视觉元素（明星肖像、收益数字等）

Step 2: 基于提取内容进行RAG审核
  - 将OCR结果作为query进行向量检索
  - 多模态模型综合图文进行合规判断
```

### 3.2 置信度计算 (`confidence_calculator.py`)

```python
# 双重置信度机制
LLM置信度：模型对判断结果的主观置信度
计算置信度：基于检索分数和违规数量的数学公式

# 融合公式
final_confidence = 0.6 × llm_confidence + 0.4 × calculated_confidence
```

### 3.3 条文切分策略

```python
# 按法规条文切分，保持语义完整性
ARTICLE_PATTERN = re.compile(r"^(第[一二三四五六七八九十百零〇0-9]+条)")

# 为什么这样切？
- 监管文档条文独立，切分后检索精度高
- 保持条文完整，避免上下文丢失
- 支持长条文自动分片（max_chunk_chars=700）
```

## 六、常见面试问题准备

### Q1: 为什么选择RAG而不是直接让大模型判断？
**答：**
1. **准确性**：监管条文需要精确引用，不能让模型"编造"
2. **可解释性**：RAG可以展示检索到的具体条文，便于审计
3. **可更新性**：法规更新时只需重建知识库，不需要重新训练模型
4. **成本**：RAG可以将长上下文压缩为相关条文，减少token消耗

### Q2: 混合检索的优势是什么？
**答：**
- **稠密检索**：擅长语义理解，如"保证收益"和"保本保息"语义相近
- **稀疏检索**：擅长精确匹配，如条文编号、专业术语
- **融合效果**：召回率提升约15-20%，特别是在专业术语场景下

### Q3: 如何处理长文档？
**答：**
1. **切分策略**：按条文边界切分，保持语义完整
2. **检索召回**：Top-K召回相关条文（默认K=6）
3. **多模态**：PDF使用VL模型进行OCR提取

### Q4: 置信度是如何计算的？
**答：**
- **LLM置信度**：模型对判断结果的主观置信
- **计算置信度**：基于检索分数和违规数量的数学公式
- **融合置信度**：`0.6 × LLM + 0.4 × 计算`

### Q5: 如果模型输出JSON格式错误怎么办？
**答：**
```python
# 鲁棒性处理
def _extract_json_block(text: str) -> dict:
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        # 正则提取JSON块
        match = re.search(r"\{.*\}", text, flags=re.DOTALL)
        if match:
            return json.loads(match.group(0))
    raise ValueError("无法解析JSON")
```

### Q6: 如何扩展支持更多监管文档？
**答：**
1. 将新文档放入 `references/` 目录
2. 重新运行 `build-kb` 命令
3. 知识库自动重建，无需修改代码

### Q7: 系统的性能瓶颈在哪里？
**答：**
1. **I/O瓶颈**：大模型API调用延迟（通过并发可优化）
2. **检索瓶颈**：向量检索复杂度O(n)，可用Milvus加速
3. **优化方向**：缓存常见查询、批量审核并发处理
